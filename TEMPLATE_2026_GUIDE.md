# 北京邮电大学 2026 届本科论文主体模板说明

本说明文档面向以下两类使用者：

1. 想直接使用本仓库撰写北京邮电大学本科毕业设计（论文）主体部分的同学。
2. 已经有 Word 稿件，希望借助 coding agent 将内容迁移到 \LaTeX{} 模板中的同学。

本仓库是一个“主体版模板”。它重点覆盖：

- 中英文摘要与关键词
- 目录
- 五章正文结构
- 参考文献
- 致谢
- 附录
- 攻读学位期间取得的创新成果

本仓库**不直接提供**你的个人封面、诚信声明、任务书、开题报告、中期检查表、成绩评定表、外文资料原文/译文等个人材料。若需要，可由用户自行从学校 Word 模板导出 PDF 后放入 `docs/` 目录。

## 1. 已确认的 2026 模板要求

以下要求来自对学校 `docx` 模板及其批注内容的提取和整理，适合作为主体排版时的工作基线。

### 1.1 页面设置

- 纸张：A4
- 页边距：上 `2.5cm`、下 `2.5cm`、左 `2.5cm`、右 `2.5cm`
- 页眉：`1.5cm`
- 页脚：`1.5cm`
- 栏数：单栏

### 1.2 页眉页脚与页码

- 页眉从正文开始显示
- 页眉内容：`北京邮电大学本科毕业设计（论文）`
- 页码位置：页面底端居中
- 目录页码：大写罗马数字 `I, II, III, ...`
- 正文页码：阿拉伯数字 `1, 2, 3, ...`
- 封面、诚信声明、论文使用授权说明、中英文摘要页通常不显示页码

### 1.3 标题与正文

- 一级标题：黑体，三号，加粗，居中，`1.5` 倍行距
- 二级标题：黑体，四号，加粗，左对齐，`1.5` 倍行距
- 三级标题：黑体，小四，加粗，左对齐，`1.5` 倍行距
- 正文：中文与中文符号用宋体小四；西文与数字用 Times New Roman 小四
- 正文段落：两端对齐，首行缩进 `2` 个中文字符，段前段后 `0` 行，`1.5` 倍行距
- 除致谢外，正文中尽量避免使用“我”“我们”等第一人称代词

### 1.4 摘要与关键词

- 中文摘要标题：`摘要`
- 英文摘要标题：`ABSTRACT`
- 中文关键词：关键词之间间隔一格，段末不加标点，关键词一般 `3--5` 个
- 英文摘要中，首次出现的重要缩写建议先写全称，再在括号中给出缩写

### 1.5 目录

- 目录标题使用三号黑体
- 一级目录项一般为黑体小四
- 二、三级目录项一般为宋体小四

### 1.6 图、表、公式

- 图题：中文楷体五号，西文 Times New Roman，居中，位于图下方
- 表题：中文楷体五号，西文 Times New Roman，居中，位于表上方
- 图号、表号按章编号，例如：`图 2-1`、`表 4-3`
- 图表上下与正文建议保持合适留白
- 公式居中，编号右对齐，按章编号，例如：`(3-1)`
- 公式后应对变量和符号作必要解释

### 1.7 参考文献

- 中文：宋体五号
- 西文与标点：Times New Roman 五号
- 两端对齐
- 悬挂缩进：`0.6cm`
- 段前段后：`0` 行
- 行距：`1.5` 倍
- 排列顺序：按正文首次引用顺序
- 数量要求：不少于 `20` 篇
- 近三年文献比例：不少于 `30%`
- 期刊用 `[J]`，会议用 `[C]`，书籍用 `[M]`，专利用 `[P]`，标准可用 `[S]`

### 1.8 其他结构性要求

- 各章通常从奇数页开始
- 第二章到第四章建议设置“本章小结”
- 本科论文通常不建议使用四级标题
- 论文中的图、表、公式应在正文中被明确引用

## 2. 本仓库当前定位

本仓库是“主体版模板”，做了如下取舍：

- 保留主体论文写作所需的核心文件与样式
- 删除旧版示例中的任务书、成绩评定表、开题报告、中期检查表、外文资料原文/译文等非主体部分
- 允许用户通过 `docs/cover.pdf` 与 `docs/statement.pdf` 自行接入学校 Word 模板导出的前置页 PDF

如果 `docs/` 目录中不存在这些 PDF，模板会自动跳过导入，不影响主体部分编译。

## 3. 如何编译

### 3.1 推荐环境

- Windows + TeX Live
- 编辑器任选：TeXworks / VS Code + LaTeX Workshop / 其他支持 XeLaTeX 的编辑器

### 3.2 基本编译顺序

在仓库根目录下，按以下顺序执行：

```powershell
xelatex main.tex
bibtex main
xelatex main.tex
xelatex main.tex
```
或者直接让你的coding-agent帮你执行

### 3.3 如果像作者一样使用绝对路径调用 TeX Live

若 `xelatex` / `bibtex` 没有加入 PATH，可直接使用 TeX Live 的绝对路径，例如：

```powershell
E:\texlive\2026\bin\windows\xelatex.exe main.tex
E:\texlive\2026\bin\windows\bibtex.exe main
E:\texlive\2026\bin\windows\xelatex.exe main.tex
E:\texlive\2026\bin\windows\xelatex.exe main.tex
```

如果你不确定路径，可以在 PowerShell 中先查看：

```powershell
$env:PATH
```

若其中包含类似 `E:\texlive\2026\bin\windows`，就可以直接调用对应可执行文件。

## 4. 典型使用方式

### 4.1 直接写 \LaTeX{}

适合已经习惯用 \LaTeX{} 写论文的同学。你主要会编辑：

- `main.cfg`
- `abstract.cfg`
- `chapter1.tex` 至 `chapter5.tex`
- `ref.bib`

### 4.2 Word 稿件迁移到 \LaTeX{}

如果你已经在 Word 中写好了大部分内容，也可以把本模板当成一个“结构容器”，逐章把内容迁移进来。建议顺序是：

1. 先填题目、摘要、关键词
2. 再按章节填正文
3. 再整理图表与公式
4. 最后统一整理参考文献和附录

## 5. 推荐的 coding-agent 使用方式

### 5.1 推荐工作流

作者个人更推荐的工作流是：

`Word / 草稿内容 + 本模板 + coding-agent 协助改格式 / 填内容 / 做一致性检查`

换句话说，本模板的一个推荐使用方式不是完全手写 \LaTeX{}，而是：

- 先准备好内容
- 再让 coding-agent 协助把内容放进模板
- 再让 coding-agent 帮你检查格式、目录、引用和版式一致性

### 5.2 OpenCode 示例配置

仓库根目录中可以放置一个示例配置文件：

- `opencode-example.json`

这是作者推荐的一种食用方案：

- 模板 + coding-agent 直接改格式
- 推荐可尝试 OpenCode 接入 `deepseek-v4-pro`
- 已经用 Word 写好的同学，也可以尝试让 coding-agent 帮忙把内容填到 \LaTeX{} 模板里

### 5.3 一个典型 prompt 思路

例如，你可以对 coding-agent 说：

```text
这是我的本科论文模板，请帮我把已经写好的中文摘要、英文摘要和第五章内容填进去；
不要改动版式要求，只做必要的结构适配；
如果发现术语、引用或格式不一致，请一并指出。
```
