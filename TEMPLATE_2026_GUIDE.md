# 北京邮电大学 2026 届本科论文主体模板说明

本说明文档面向以下两类使用者：

1. 想直接使用本仓库撰写北京邮电大学本科毕业设计（论文）主体部分的同学。
2. 已经有 Word 稿件，希望借助 coding agent 将内容迁移到 \LaTeX{} 模板中的同学。

本仓库是一个“主体版模板”。它重点覆盖：

- 中英文摘要与关键词
- 目录
- 五章正文结构
- 参考文献
- 致谢
- 附录
- 攻读学位期间取得的创新成果

本仓库**不直接提供**你的个人封面、诚信声明、任务书、开题报告、中期检查表、成绩评定表、外文资料原文/译文等个人材料。若需要，可由用户自行从学校 Word 模板导出 PDF 后放入 `docs/` 目录。

## 1. 已确认的 2026 模板要求

以下要求来自对学校 `docx` 模板及其批注内容的提取和整理，适合作为主体排版时的工作基线。

### 1.1 页面设置

- 纸张：A4
- 页边距：上 `2.5cm`、下 `2.5cm`、左 `2.5cm`、右 `2.5cm`
- 页眉：`1.5cm`
- 页脚：`1.5cm`
- 栏数：单栏

### 1.2 页眉页脚与页码

- 页眉从正文开始显示
- 页眉内容：`北京邮电大学本科毕业设计（论文）`
- 页码位置：页面底端居中
- 目录页码：大写罗马数字 `I, II, III, ...`
- 正文页码：阿拉伯数字 `1, 2, 3, ...`
- 封面、诚信声明、论文使用授权说明、中英文摘要页通常不显示页码

### 1.3 标题与正文

- 一级标题：黑体，三号，加粗，居中，`1.5` 倍行距
- 二级标题：黑体，四号，加粗，左对齐，`1.5` 倍行距
- 三级标题：黑体，小四，加粗，左对齐，`1.5` 倍行距
- 正文：中文与中文符号用宋体小四；西文与数字用 Times New Roman 小四
- 正文段落：两端对齐，首行缩进 `2` 个中文字符，段前段后 `0` 行，`1.5` 倍行距
- 除致谢外，正文中尽量避免使用“我”“我们”等第一人称代词

### 1.4 摘要与关键词

- 中文摘要标题：`摘要`
- 英文摘要标题：`ABSTRACT`
- 中文关键词：关键词之间间隔一格，段末不加标点，关键词一般 `3--5` 个
- 英文摘要中，首次出现的重要缩写建议先写全称，再在括号中给出缩写

### 1.5 目录

- 目录标题使用三号黑体
- 一级目录项一般为黑体小四
- 二、三级目录项一般为宋体小四

### 1.6 图、表、公式

- 图题：中文楷体五号，西文 Times New Roman，居中，位于图下方
- 表题：中文楷体五号，西文 Times New Roman，居中，位于表上方
- 图号、表号按章编号，例如：`图 2-1`、`表 4-3`
- 图表上下与正文建议保持合适留白
- 公式居中，编号右对齐，按章编号，例如：`(3-1)`
- 公式后应对变量和符号作必要解释

### 1.7 参考文献

- 中文：宋体五号
- 西文与标点：Times New Roman 五号
- 两端对齐
- 悬挂缩进：`0.6cm`
- 段前段后：`0` 行
- 行距：`1.5` 倍
- 排列顺序：按正文首次引用顺序
- 数量要求：不少于 `20` 篇
- 近三年文献比例：不少于 `30%`
- 期刊用 `[J]`，会议用 `[C]`，书籍用 `[M]`，专利用 `[P]`，标准可用 `[S]`

### 1.8 其他结构性要求

- 各章通常从奇数页开始
- 第二章到第四章建议设置“本章小结”
- 本科论文通常不建议使用四级标题
- 论文中的图、表、公式应在正文中被明确引用

### 1.9 写作时的字体替换指南

模板本身不会自动清理你正文里的数学写法。如果你的章节中仍在大量使用传统的 `$...$` 数学模式，学校的格式检测工具仍然可能报出 `CMR12` / `CMMI12` 等"非 Times New Roman"字体警告。

因此，在实际写作时，**需要逐章将高频出现的参数记号、信噪比、科学计数法、区间范围等写法手动替换为模板提供的文本字体宏**。以下表格是本次整理论文中经过检测验证的替换对照：

| 想表达的内容 | 旧写法（可能触发检测） | 推荐写法（检测安全） |
|---|---|---|
| 单个变量名 | `$N$`、`$A$`、`$K$` | `\tnrvar{N}`、`\tnrvar{A}`、`\tnrvar{K}` |
| 带下标的变量 | `$G_N$`、`$x_i$`、`$L_{CRC}$` | `\tnrsub{G}{N}`、`\tnrsub{x}{i}`、`\tnrsubrm{L}{CRC}` |
| 等号关系 | `$K=A+L_{CRC}$` | `\tnrvar{K}=\tnrvar{A}+\tnrsubrm{L}{CRC}` |
| 信噪比 | `$E_b/N_0$` | `\EbNZero` |
| 科学计数法 | `$5.20\times10^{-3}$` | `\scinum{5.20}{-3}` |
| 区间范围 | `$[-31,31]$` | `\tnrrange{-31}{31}` |
| dB 单位 | `$\mathrm{dB}$` 或 `\,dB` | 直接写 `dB`（正文文本） |

几点说明：

- 以上替换**不是必须 100% 逐处覆盖**，但在检测报告中反复出现的点位（如每个章节里大量出现的 `N=`、`A=`、`E_b/N_0`、科学计数法）**建议优先替换**。
- 普通的短文本（如 `5G`、`Wi-Fi`、`FPGA`）直接写即可，不需要放进数学模式。
- 公式中的希腊字母（如 `\sigma`）和矩阵排版（如 `\begin{bmatrix}`）一般不会被检测特别针对，如果检测器仍报了，可根据具体情况改用正文路线或保留下节中的矩阵注意事项。
- 每次做完替换后，建议重新编译并跑一次学校格式检测，不要只看 PDF 的肉眼效果。

这些宏的定义均位于 `BUPTthesisbachelor.sty`，目前已经随模板一起提供。

**关于简单 `$...$` 写法的额外说明**：模板已经从 `mathastext` 层面做了强化处理（`BUPTthesisbachelor.sty:70-83`，由 [@yuchenlogin](https://github.com/yuchenlogin) 贡献），数学模式中的数字 `0-9` 和常见半角符号（`=`、`+`、`-`、`(`、`)`、`[`、`]` 等）会直接被映射到正文数学字体。这使得 `$N=1024$`、`$A=256$` 这类简单的参数表达**有较大概率直接通过检测**，不再强制要求替换为文本宏。但复杂公式中的希腊字母、矩阵符号和大型运算符仍可能由数学字体包提供，如果检测报告继续定位到这类内容，再按上表改为正文式记号。

本地提交学校检测前，建议先用 `pdffonts` 做一次快速自查。若输出为空，说明当前 PDF 中没有嵌入常见 Computer Modern 字体：

```bash
for p in $(seq 1 50); do
  pdffonts -f "$p" -l "$p" main.pdf | grep -E "CMR|CMMI|CMSY|CMEX" && echo "page $p"
done
```

## 2. 本仓库当前定位

本仓库是“主体版模板”，做了如下取舍：

- 保留主体论文写作所需的核心文件与样式
- 删除旧版示例中的任务书、成绩评定表、开题报告、中期检查表、外文资料原文/译文等非主体部分

## 3. 如何编译

### 3.1 推荐环境

- Windows + TeX Live
- 编辑器任选：TeXworks / VS Code + LaTeX Workshop / 其他支持 XeLaTeX 的编辑器

### 3.2 基本编译顺序

在仓库根目录下，按以下顺序执行：

```powershell
xelatex main.tex
bibtex main
xelatex main.tex
xelatex main.tex
```
或者直接让你的coding-agent帮你执行

### 3.3 如果像作者一样使用绝对路径调用 TeX Live

若 `xelatex` / `bibtex` 没有加入 PATH，可直接使用 TeX Live 的绝对路径，例如：

```powershell
E:\texlive\2026\bin\windows\xelatex.exe main.tex
E:\texlive\2026\bin\windows\bibtex.exe main
E:\texlive\2026\bin\windows\xelatex.exe main.tex
E:\texlive\2026\bin\windows\xelatex.exe main.tex
```

如果你不确定路径，可以在 PowerShell 中先查看：

```powershell
$env:PATH
```

若其中包含类似 `E:\texlive\2026\bin\windows`，就可以直接调用对应可执行文件。

### 5.4 让 coding-agent 帮你自动批量替换数学字体写法

如果你已经用传统的 `$...$` 数学模式写完了一整章内容，可以直接把下面的指令输入给 coding-agent，让它扫描并替换所有容易触发学校格式检测的写法。以下是一份通用的 agent 指令模板：

```text
这是我的本科论文模板，模板的 .sty 文件中已经预定义了以下宏，
用于把英文字母、数字和常见符号输出为 Times New Roman：

  \tnrvar{<字母>}           单个意大利体变量（如 N, A, K, x, y）
  \tnrsub{<主字母>}{<下标>}  带意大利体下标的变量（如 x_i, G_N）
  \tnrsubrm{<主字母>}{<下标>}带正体罗马下标的变量（如 L_CRC）
  \scinum{<系数>}{<指数>}    科学计数法（如 5.20×10⁻³）
  \tnrrange{<下限>}{<上限>}  闭区间范围（如 [-31,31]）

现在请你对 <文件列表> 中的每一个 .tex 文件，
按以下规则进行一次系统性搜索和替换：

1. 扫描所有 $...$ 内仅包含单个大写或小写拉丁字母的变量名，
   替换为 \tnrvar{<字母>}。
   注意：$ 中包含运算符、多个字符、希腊字母或下标时，
   不要整体替换，按后续规则拆分处理。

2. 扫描 $...$ 内形如 <单字母>_{<内容>} 的下标变量：
   - 如果下标是普通意大利体字母/数字，替换为 \tnrsub{<主字母>}{<下标>}；
   - 如果下标是罗马体缩写（全部大写或多字母），
     替换为 \tnrsubrm{<主字母>}{<下标>}。

3. 扫描 $...$ 内形如 <数字>.<数字>\times10^{<指数>} 的科学计数法，
   替换为 \scinum{<系数>}{<指数>}。

4. 扫描 $...$ 内形如 [-<数字>,<数字>] 的闭区间范围，
   替换为 \tnrrange{<下限>}{<上限>}。

5. 对于论文中特有的多字符专业记号（如信噪比等），
   检查模板 .sty 中是否已有对应宏；如果没有，
   可以在 main.tex 导言区用类似方式自行定义（\newcommand{\MyNotation}{...}），
   然后再在正文中替换旧记号。

6. 正文中包裹在 \mathrm{} 里的单位（如 \mathrm{dB}）直接去掉 \mathrm，
   改为普通正文文本（如 dB），无需额外命令。

7. 替换完毕后，
   - 对每个文件逐条输出"原写法 → 改后写法"的变更记录；
   - 用 xelatex 编译一次，确认没有因替换产生新的语法错误。

注意事项：
- 不要在 \begin{equation} ... \end{equation} 内部的希腊字母、
  矩阵和复杂多行公式中盲目替换；
- 如果原写法中有多字符连续的情况（如 $K=A+L_{CRC}$），先拆分成
  单个变量/运算符，再替换变量部分；
- 全程不要改动 .sty 文件、main.tex 的结构和 \bibliography 命令。
```

把这个指令直接发给 coding-agent，并指定要处理的文件（如 `chapter1.tex chapter2.tex chapter3.tex`），它就会批量扫描并替换。

**替换后必须手工复查——agent 最容易漏掉的 5 类残留**：

agent 按规则替换完以后，章节里往往还会残留以下写法，它们同样会触发 `CMR12` / `CMMI12` 报错，需要逐条搜索确认：

| 残留写法 | 为什么会被漏掉 | 改为 |
|---|---|---|
| `$0.5$`、`$2.5$`、`$3.0$` 等裸数字 | 规则里只写了单字母变量，纯数字不在匹配范围 | 模板已内置数学数字强化，这类通常不会触发检测；若检测器仍报，改为正文 `0.5`、`2.5` |
| `$10^{-1}$`、`$10^{-3}$`（没有 `\times` 的幂次） | 不匹配"科学计数法"的 `\times` 模式 | `\scinum{10}{-1}` 或拆成 `10\textsuperscript{-1}` |
| `$\mathrm{dB}$` | `\mathrm` 包裹，agent 可能跳过 | `dB`（直接正文） |
| `$2L$`、`$4L$` 等数字+字母组合 | 不是"单个字母"，也不匹配下标/科学计数法规则 | `2\tnrvar{L}`（数字已会被强化，仅字母部分需改） |
| `$\log_2 N$`、`$\min(...)$` 等运算符 | 规则说不要动公式内部，agent 会跳过 | 保持原样（这类通常不会被检测重点针对；如果报了再改） |

**复查方法**——注意模板已经内置了数学数字强化，`$0.5$`、`$2.5$` 等纯数字项通常不再触发检测。重点排查以下仍可能残留的写法：

```powershell
Select-String -Path "chapter*.tex" -Pattern '\$[^$]+\$' | Select-Object -First 80
```

检查输出列表里是否还有上述 5 类残留。每清完一类，重新编译一次 `main.pdf` 并跑学校检测器验证。

**如果复查后检测器仍然报**，说明有更深层的数学字形路径没有被 `mathastext` + 文本宏覆盖到。此时需要把检测报告的具体报错原文贴出来，对照定位是哪个公式/变量还在走旧数学字体。

## 6. 调试记录

这一节记录了模板在 2026 届本科论文排版过程中，已经踩过并验证过的几个高频问题。它的目的不是替代学校要求，而是帮助后来者少走弯路。

### 6.1 参考文献明明设置成五号，检测仍报小四

现象：

- 参考文献正文被格式检测工具判定为 `小四(12.0磅)`，而不是要求的 `五号(10.5磅)`。

常见误区：

- 只在样式文件里定义 `\refbodyfont`
- 只在 `thebibliography` 环境开始处调用 `\refbodyfont`
- 只改 `\bibfont`，但不检查 `natbib`、列表环境和 `\bibitem` 是否在后面把字号重置掉

根因：

- `natbib` 的参考文献环境会经过自己的列表初始化过程。
- 仅在环境开头设置字号，往往会被后续列表命令重新带回正文默认字号。

最终稳定方案：

- 在 `main.tex` 的参考文献前，重定义 `\bibnumfmt` 与 `\bibitem`。
- 编号标签 `[1]` 与每条文献正文都分别强制设置为 `宋体 + 五号(10.5pt) + 1.5 倍行距`。
- 关键顺序是：先 `\setstretch{1.5}`，再 `\fontsize{10.5pt}{15.75pt}\selectfont`；若顺序相反，字号可能又被拉回 `12pt`。

模板中的落点：

- `main.tex` 中参考文献调用前的 `\bibnumfmt` / `\bibitem` 重定义。

### 6.2 数学中的英文字母、数字和半角标点被检测为非 Times New Roman

现象：

- `E_b/N_0`、`N=1024`、`A=256`、`K=A+L_{CRC}`、公式中的拉丁字母、数字、括号、等号、斜杠等，被检测器报成 `CMR12`、`CMMI12` 或其他非 Times New Roman 字体。

常见误区：

- 直接上 `unicode-math` 或其他数学字体包，希望“一次性统一所有数学字体”。
- 这样虽然能消掉 Computer Modern，但检测器可能继续报成 `TeXGyreTermesMath` 等“仍然不是新罗马”的字体名。

最终稳定思路：

- 对于学校检测更敏感的拉丁字母、数字和半角标点，优先采用正文 Times 路线，而不是完全依赖独立数学字体族。
- 模板里使用 `mathastext`，让数学中的英文、数字和常见标点尽量走正文 Times New Roman。
- 对极易触发检测的问题写法，额外提供文本宏，例如：
  - `\tnrvar`
  - `\tnrsub`
  - `\tnrsubrm`
  - `\EbNZero`
  - `\scinum`
  - `\tnrrange`

适用场景：

- 参数记号
- 信噪比记号
- 科学计数法
- 区间范围写法

### 6.3 科学计数法会把数字重新带回 CMR12

现象：

- 类似 `5.20\times10^{-3}`、`2.60\times10^{-6}` 的写法，即使前后文字都正常，检测器仍可能把其中数字识别为 `CMR12`。

原因：

- 这类写法仍然处在传统数学模式中，`\times`、上标数字和 `10` 的组合容易重新调用数学字体。

最终稳定方案：

- 使用正文式科学计数法宏 `\scinum{5.20}{-3}`。
- 其输出使用正文 Roman 字体和文本上标，避免重新走旧的数学字形路径。

### 6.4 区间写法 `[-31,31]` 里的括号和数字也可能触发检测

现象：

- 定点量化范围、节点范围等写法中的方括号和数字仍可能被判成数学字体。

最终稳定方案：

- 使用正文宏 `\tnrrange{-31}{31}`。
- 让方括号、逗号、负号和数字都走正文 Roman 路线。

### 6.5 矩阵公式被“压平”成一行怎么办

现象：

- 若为了强制使用正文 Times，把整个矩阵都包进 `\text{...}`，矩阵往往会失去二维结构，被压扁成一行文本。

最终稳定方案：

- 拉丁变量名可以文本化，但矩阵本体仍应保留真实矩阵结构，例如继续使用 `array` / `bmatrix` / `\left[ ... \right]` 这一类二维公式环境。
- 不要为了字体检测，把“需要二维排版”的对象整体塞进纯文本命令。

### 6.6 一个实用建议

- 如果你已经解决了检测问题，不要轻易再改动参考文献或数学字体相关底层代码。
- 每次修改后，优先重新跑学校检测工具，而不是只看 PDF 肉眼效果。
- 肉眼上“看起来像五号”或“看起来像新罗马”，并不代表学校检测器一定会认。
